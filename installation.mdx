## Overview

Softbee is a cross-platform Flutter application that runs on Android, iOS, Web, Windows, macOS, and Linux. This guide covers installation and setup for all supported platforms.

<Note>
  **Requirements**: 
  - Flutter SDK 3.10.7 or higher
  - Dart SDK (included with Flutter)
  - Platform-specific development tools
</Note>

## Quick setup

Get Softbee running on your development machine in minutes.

<Steps>

<Step title="Clone the repository">

Clone the Softbee repository from GitHub:

```bash
git clone https://github.com/SoffiaSanchezz/Softbee-App.git
cd Softbee
```

</Step>

<Step title="Configure environment variables">

Create a `.env` file in the project root directory with your API credentials:

```bash .env
# Copy from .env.example
API_BASE_URL=https://your-api-endpoint.com
API_KEY=your_api_key_here
```

<Warning>
  Never commit your `.env` file to version control. It's already included in `.gitignore`.
</Warning>

</Step>

<Step title="Install dependencies">

Install all required Flutter packages:

```bash
flutter pub get
```

The project uses the following key dependencies:

- `flutter_riverpod: ^2.6.1` - State management
- `go_router: ^17.0.1` - Navigation and routing
- `dio: ^5.3.3` - HTTP client for API requests
- `shared_preferences: ^2.2.2` - Local data persistence
- `flutter_secure_storage: ^10.0.0` - Secure credential storage
- `lottie: ^2.7.0` - Smooth animations
- `google_fonts: ^8.0.0` - Custom typography

</Step>

<Step title="Generate code (if applicable)">

If the project has code generation requirements, run:

```bash
dart run build_runner build --delete-conflicting-outputs
```

</Step>

</Steps>

## Platform-specific installation

<Tabs>
  <Tab title="Android">
    
### Android Setup

<Steps>

<Step title="Install Android Studio">

1. Download and install [Android Studio](https://developer.android.com/studio)
2. Open Android Studio and install the Android SDK
3. Configure the Android SDK path in Flutter:

```bash
flutter doctor --android-licenses
```

</Step>

<Step title="Set up an emulator or device">

**Option A: Use an Android emulator**
1. Open Android Studio
2. Go to **Tools** > **Device Manager**
3. Click **Create Device**
4. Select a device definition (e.g., Pixel 5)
5. Download and select a system image (API 30+)
6. Click **Finish** to create the emulator

**Option B: Use a physical device**
1. Enable **Developer Options** on your Android device
2. Enable **USB Debugging**
3. Connect your device via USB
4. Run `flutter devices` to verify connection

</Step>

<Step title="Run the app">

Launch Softbee on Android:

```bash
flutter run
# Or specify the device
flutter run -d <device-id>
```

<Tip>
  Use `flutter run --release` for a production build with better performance.
</Tip>

</Step>

</Steps>

### Build APK

To create an installable APK:

```bash
flutter build apk --release
```

The APK will be located at `build/app/outputs/flutter-apk/app-release.apk`.

### Build App Bundle

For Google Play Store submission:

```bash
flutter build appbundle --release
```

### iOS Setup

<Note>
  iOS development requires a Mac with Xcode installed.
</Note>

<Steps>

<Step title="Install Xcode">

1. Download Xcode from the [Mac App Store](https://apps.apple.com/us/app/xcode/id497799835)
2. Install Xcode Command Line Tools:

```bash
sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer
sudo xcodebuild -runFirstLaunch
```

3. Accept the Xcode license:

```bash
sudo xcodebuild -license accept
```

</Step>

<Step title="Install CocoaPods">

CocoaPods manages iOS dependencies:

```bash
sudo gem install cocoapods
```

</Step>

<Step title="Set up iOS Simulator or device">

**Option A: Use iOS Simulator**
1. Open Xcode
2. Go to **Xcode** > **Preferences** > **Components**
3. Download an iOS Simulator
4. Run `open -a Simulator` to launch

**Option B: Use a physical device**
1. Connect your iPhone/iPad via USB
2. Trust the computer on your device
3. Set up your Apple Developer account in Xcode

</Step>

<Step title="Run the app">

Navigate to the iOS directory and install pods:

```bash
cd ios
pod install
cd ..
```

Launch Softbee on iOS:

```bash
flutter run
# Or specify the device
flutter run -d <device-id>
```

</Step>

</Steps>

### Build for iOS

To create an iOS build:

```bash
flutter build ios --release
```

<Warning>
  iOS app distribution requires an Apple Developer account ($99/year).
</Warning>

### Web Setup

Softbee runs in modern web browsers with Flutter Web support.

<Steps>

<Step title="Enable web support">

Ensure Flutter web is enabled:

```bash
flutter config --enable-web
```

</Step>

<Step title="Run in development mode">

Start the development server:

```bash
flutter run -d chrome
# Or use other browsers
flutter run -d edge
flutter run -d web-server
```

Access the app at `http://localhost:5000` (or the port shown in terminal).

</Step>

<Step title="Build for production">

Create an optimized web build:

```bash
flutter build web --release
```

The build output will be in `build/web/`.

</Step>

</Steps>

### Deploy to hosting

<CodeGroup>

```bash Firebase Hosting
# Install Firebase CLI
npm install -g firebase-tools

# Login and initialize
firebase login
firebase init hosting

# Deploy
firebase deploy --only hosting
```

```bash Netlify
# Install Netlify CLI
npm install -g netlify-cli

# Navigate to build directory
cd build/web

# Deploy
netlify deploy --prod
```

```bash Vercel
# Install Vercel CLI
npm install -g vercel

# Navigate to build directory
cd build/web

# Deploy
vercel --prod
```

</CodeGroup>

<Tip>
  For web deployment, ensure your API endpoints support CORS (Cross-Origin Resource Sharing).
</Tip>

## Development workflow

### Project structure

Softbee follows Clean Architecture principles:

```
lib/
├── core/                  # Global configuration and utilities
│   ├── error/            # Error handling and exceptions
│   ├── network/          # Dio HTTP client setup
│   ├── router/           # GoRouter navigation configuration
│   ├── theme/            # App colors, fonts, and styles
│   └── widgets/          # Shared UI components
├── feature/              # Feature modules (Screaming Architecture)
│   ├── auth/            # Authentication and user management
│   │   ├── core/        # Domain entities and use cases
│   │   ├── data/        # Repositories and data sources
│   │   └── presentation/# UI pages, widgets, and state
│   ├── apiaries/        # Apiary management
│   ├── beehive/         # Beehive tracking
│   ├── monitoring/      # Production monitoring
│   └── inventory/       # Equipment and supplies
└── main.dart            # Application entry point
```

### Running tests

```bash
# Run all tests
flutter test

# Run with coverage
flutter test --coverage

# Run specific test file
flutter test test/feature/auth/login_test.dart
```

### Code quality

```bash
# Analyze code
flutter analyze

# Format code
dart format .

# Check for outdated packages
flutter pub outdated
```

## Troubleshooting

<AccordionGroup>

<Accordion title="Flutter doctor shows issues" icon="stethoscope">

Run `flutter doctor -v` to see detailed diagnostic information:

```bash
flutter doctor -v
```

Common fixes:
- **Android licenses**: Run `flutter doctor --android-licenses`
- **Xcode**: Update to the latest version from App Store
- **CocoaPods**: Run `sudo gem install cocoapods`

</Accordion>

<Accordion title="Build fails with dependency errors" icon="triangle-exclamation">

Clean and reinstall dependencies:

```bash
flutter clean
flutter pub get
```

For iOS specifically:

```bash
cd ios
rm -rf Pods Podfile.lock
pod install
cd ..
```

</Accordion>

<Accordion title="Hot reload not working" icon="fire">

Restart the development server:

```bash
# Press 'r' in terminal for hot reload
# Press 'R' for hot restart
# Press 'q' to quit
```

If issues persist, stop and restart:

```bash
flutter run
```

</Accordion>

<Accordion title="App crashes on startup" icon="bug">

Check for missing environment variables:

1. Verify `.env` file exists and contains required values
2. Ensure API endpoints are accessible
3. Check device/emulator logs:

```bash
# Android
adb logcat

# iOS
xcrun simctl spawn booted log stream
```

</Accordion>

</AccordionGroup>
